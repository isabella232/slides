The WebRTC Data Channel is a High Performance, Low Latency Connection to Other Browsers

Browser A
Browser B

We at PubNub want to solve this issue for developers by building the core building blocks for real time applications.

Capabilities

  • Allows sending of ArrayBuffers, Blobs, and Strings
  • Works in reliable or unreliable modes
  • Reliable works much like TCP, guaranteed order and no packet loss
  • Un-Reliable works much like UDP, no guarantees
What does this solution look like? Well the best solution is something that can support both unicast and broadcast systems.

Demo

Let's check out an example of this in action.

How Does it Work?

var peerConnection = new RTCPeerConnection();

var dataChannel = peerConnection.createDataChannel("myLabel");

dataChannel.onerror = function (err) {
  console.log("Channel Error:", err);
};

dataChannel.onmessage = function (event) {
  console.log("Got Message:", event.data);
};

dataChannel.send("Hello World!");
And now how is this built on our system? It is simple: pub and sub. We do the rest!

Is it Ready?

  • Ready for Chrome and Firefox Today
  • Ready for Android Chrome and Firefox Today
  • Interoperability between browsers still being worked on (but soon!)
  • IE, Safari, and Opera still unknown

Security

The data channel is encrptyed from browser to browser so data cannot be stolen. On the other hand, the biggest threat is your users sending malicious data to another user.

Other Pitfalls
